## Gotchas / do-not-break
- `server/db.js` — do NOT commit unrelated changes; it holds the org migration ladder.
- Coolify REST omits: env VALUES, build logs, deploy history (finished), destinations, project name on apps → all read from Coolify's DB over SSH (`coolifydb.js`).
- `application_deployment_queues.application_id` is **varchar** (cast `id::text`); volumes' `resource_id` is bigint.
- Never let a `useEffect` callback return a Promise.
